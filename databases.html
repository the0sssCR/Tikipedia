Ба́за да́нных (БД; англ. database) — организованная совокупность данных, предназначенная для хранения, обработки и извлечения информации. Базы данных управляются при помощи систем управления базами данных (СУБД)[1].

Концепция баз данных возникла в 1960-х годах. В 1970 году Эдгар Кодд предложил реляционную модель данных, которая стала доминирующей в индустрии и остаётся таковой по сей день[2].

Определение

База данных — это совокупность взаимосвязанных данных, организованных в соответствии с определённой моделью данных. Данные в БД хранятся таким образом, чтобы обеспечить их целостность, непротиворечивость и эффективный доступ.

Типы баз данных

Существует несколько основных типов баз данных:

Реляционные — данные организованы в таблицы (отношения) с заранее определённой схемой. Примеры СУБД: PostgreSQL, MySQL, Oracle[3].

Документоориентированные — данные хранятся в виде документов (JSON, BSON). Примеры: MongoDB, CouchDB.

Ключ-значение — простейшая модель, где каждое значение ассоциировано с уникальным ключом. Примеры: Redis, DynamoDB.

Графовые — данные представлены в виде графа с узлами и рёбрами. Примеры: Neo4j, ArangoDB.

Реляционные базы данных

В реляционной модели данные представляются в виде таблиц (отношений). Каждая таблица состоит из строк (кортежей) и столбцов (атрибутов). Связи между таблицами устанавливаются через первичные и внешние ключи[4].

Основные свойства реляционной модели (свойства ACID):

• Атомарность (Atomicity) — транзакция выполняется полностью или не выполняется вовсе.

• Согласованность (Consistency) — данные переходят из одного согласованного состояния в другое.

• Изолированность (Isolation) — параллельные транзакции не влияют друг на друга.

• Долговечность (Durability) — результат завершённой транзакции сохраняется.

Язык SQL

SQL (Structured Query Language) — стандартный язык для взаимодействия с реляционными БД. Основные операции:

• SELECT — выборка данных из таблицы

• INSERT — вставка новых записей

• UPDATE — обновление существующих записей

• DELETE — удаление записей

• JOIN — объединение данных из нескольких таблиц[5]

Нормализация

Нормализация — процесс организации данных для минимизации избыточности. Основные нормальные формы:

1НФ — все атрибуты атомарны (неделимы).

2НФ — выполнена 1НФ, и все неключевые атрибуты полностью зависят от первичного ключа.

3НФ — выполнена 2НФ, и нет транзитивных зависимостей.

НФБК (Бойса—Кодда) — усиленная третья нормальная форма[6].

NoSQL

NoSQL (Not Only SQL) — класс СУБД, не использующих реляционную модель данных. NoSQL базы данных оптимизированы для работы с большими объёмами данных, горизонтального масштабирования и гибких схем. Вместо свойств ACID часто используется модель BASE (Basically Available, Soft state, Eventually consistent)[7].

Анализ данных

Анализ баз данных включает в себя комплекс методов для извлечения полезной информации из хранимых данных:

OLAP (Online Analytical Processing) — технология оперативного анализа многомерных данных.

Data Mining — обнаружение скрытых закономерностей в больших массивах данных с помощью методов машинного обучения и статистики.

ETL (Extract, Transform, Load) — процесс извлечения, преобразования и загрузки данных в хранилище данных.

Применение

Базы данных являются фундаментом современных информационных систем. Они применяются в банковском деле, электронной коммерции, здравоохранении, логистике, социальных сетях и практически любой области, связанной с обработкой информации.
